# agi.utils：基础设施与通用工具层

`agi.utils` 提供跨任务复用的基础能力，是项目工程化稳定性的关键。

## 能力清单

- `search_engine.py`：统一搜索入口与结果封装。
- `scrape.py`：网页内容抓取与正文提取。
- `yacy.py`：YaCy 搜索引擎集成。
- `nlp.py`：文本预处理、分词/停用词相关工具。
- `file_storage.py`：文件持久化与访问路径转换。
- `weather.py`、`stock_market.py`：外部信息查询工具。
- `common.py`：跨模块通用函数。

## 设计原则

- **无业务耦合**：避免引入任务编排逻辑，保持可复用性。
- **可替换实现**：对外尽量暴露稳定函数签名，底层依赖可替换。
- **可测试优先**：复杂逻辑应可通过 `tests/utils` 独立验证。

## 建议

- 新增工具优先放在 `utils`，通过 `tasks/tools.py` 再暴露给 Agent。
- 涉及外部 API 的函数要有超时、异常收敛与日志。
