CompositeBackend = 一个“虚拟文件系统路由层”，把不同路径映射到不同存储后端。
CompositeBackend 的路由前缀是由上层 Agent 开发者或 Runtime 配置者提供的，框架不会自动生成，通常根据功能或存储策略人为划分路径。


| 方法 / 属性                 | 类型                               | 参数 / 返回                                                            | 功能说明                                           | 路由 / 特殊行为                                                        |
| ----------------------- | -------------------------------- | ------------------------------------------------------------------ | ---------------------------------------------- | ---------------------------------------------------------------- |
| `__init__`              | 构造函数                             | `default: BackendProtocol`<br>`routes: dict[str, BackendProtocol]` | 初始化 CompositeBackend，设置默认 backend 和路径路由        | 路由按最长前缀匹配                                                        |
| `default`               | BackendProtocol                  | -                                                                  | 默认 backend，用于未匹配路径                             | -                                                                |
| `routes`                | dict                             | -                                                                  | 路径前缀 → backend                                 | -                                                                |
| `sorted_routes`         | list                             | -                                                                  | 按长度降序排列的路由列表                                   | 确保最长前缀优先匹配                                                       |
| `_route_for_path`       | 内部函数                             | `path`                                                             | 根据路径返回 `(backend, backend_path, route_prefix)` | 核心路由逻辑                                                           |
| `_remap_grep_path`      | 内部函数                             | `GrepMatch, route_prefix`                                          | 将 grep 结果路径恢复到原始前缀                             | -                                                                |
| `_remap_file_info_path` | 内部函数                             | `FileInfo, route_prefix`                                           | 将 FileInfo 路径恢复到原始前缀                           | -                                                                |
| `_get_backend_and_key`  | 内部函数                             | `key`                                                              | 获取路由 backend 和处理后的路径                           | -                                                                |
| `ls_info`               | list[FileInfo]                   | `path`                                                             | 列出目录内容                                         | `/` 返回 default + 所有 route 目录；匹配 route 返回对应 backend；其他只查询 default |
| `als_info`              | async list[FileInfo]             | `path`                                                             | 异步版本 `ls_info`                                 | 同上                                                               |
| `read`                  | str                              | `file_path, offset=0, limit=2000`                                  | 读取文件内容                                         | 路由到匹配 backend                                                    |
| `aread`                 | async str                        | `file_path, offset=0, limit=2000`                                  | 异步版本 `read`                                    | 同上                                                               |
| `write`                 | WriteResult                      | `file_path, content`                                               | 写文件                                            | 路由到匹配 backend，更新 runtime.state.files（如 default 是 StateBackend）   |
| `awrite`                | async WriteResult                | `file_path, content`                                               | 异步版本 `write`                                   | 同上                                                               |
| `edit`                  | EditResult                       | `file_path, old_string, new_string, replace_all=False`             | 替换文件内容                                         | 路由到匹配 backend，更新 runtime.state.files                             |
| `aedit`                 | async EditResult                 | `file_path, old_string, new_string, replace_all=False`             | 异步版本 `edit`                                    | 同上                                                               |
| `grep_raw`              | list[GrepMatch] / str            | `pattern, path=None, glob=None`                                    | 文本搜索                                           | path=None 或 `/` 搜索所有 backend，匹配 route 搜索单 backend，否则 default     |
| `agrep_raw`             | async list[GrepMatch] / str      | `pattern, path=None, glob=None`                                    | 异步版本 `grep_raw`                                | 同上                                                               |
| `glob_info`             | list[FileInfo]                   | `pattern, path="/"`                                                | glob 查找                                        | 匹配 route 搜索该 backend，否则搜索 default + 所有 routed backends           |
| `aglob_info`            | async list[FileInfo]             | `pattern, path="/"`                                                | 异步版本 `glob_info`                               | 同上                                                               |
| `upload_files`          | list[FileUploadResponse]         | `files: list[tuple[path, bytes]]`                                  | 批量上传                                           | 按 backend 分组调用，每个 backend 一次性上传                                  |
| `aupload_files`         | async list[FileUploadResponse]   | `files: list[tuple[path, bytes]]`                                  | 异步版本 `upload_files`                            | 同上                                                               |
| `download_files`        | list[FileDownloadResponse]       | `paths: list[str]`                                                 | 批量下载                                           | 按 backend 分组调用，每个 backend 一次性下载                                  |
| `adownload_files`       | async list[FileDownloadResponse] | `paths: list[str]`                                                 | 异步版本 `download_files`                          | 同上                                                               |
| `execute`               | ExecuteResponse                  | `command, timeout=None`                                            | 执行 shell 命令                                    | **仅 default backend**，要求 default 是 SandboxBackendProtocol        |
| `aexecute`              | async ExecuteResponse            | `command, timeout=None`                                            | 异步版本 `execute`                                 | 同上                                                               |
